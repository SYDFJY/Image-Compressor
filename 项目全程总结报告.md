## 一、项目概述

### 1.1 项目背景

在日常学习与工作中，图片与视频文件体积过大导致传输慢、存储占用多、网页加载延迟等问题频繁出现。市面上的在线压缩工具存在**隐私泄露风险**（文件上传至第三方服务器），专业软件（如 Photoshop、Premiere）**操作门槛高、授权费用昂贵**。本项目开发一款**离线、轻量、易用**的桌面图片/视频双模压缩工具，定位为"人人都能用的压缩工具"，面向普通用户而非专业设计师。

### 1.2 项目目标

| 目标维度 | 具体描述 |
|---------|---------|
| **核心功能** | 支持 JPG/PNG/BMP/GIF/WebP/TIFF/ICO 等格式的批量图片压缩 + 8 种视频压缩，含质量/尺寸/格式/帧率/音频/裁剪/封面提取/码率模式/目标大小 |
| **易用性** | 拖拽导入、右键菜单、全局快捷键、参数记忆、模式一键切换、3 步完成压缩、悬停大图预览、批量重命名、文件夹自动监控 |
| **用户体验** | 启动画面、Toast 通知、渐变进度条、3 种预览模式、23 套可切换主题、搜索筛选排序、主题色描边卡片、按钮/输入控件统一配色 |
| **可靠性** | 单文件失败不影响整体、中途可取消、异常全面容错、EDT 非阻塞设计、并行竞态防护、压缩大小守卫 |
| **可交付** | 绿色免安装 fat JAR + 捆绑 JRE+FFmpeg+VLC 便携版（267 MB ZIP 解压即用）+ EXE/VBS/BAT 三重启动器 |

### 1.4 技术约束（绝对红线）

1. ❌ 禁止使用 JDK 8 以上版本 API（`var`、`String.repeat()`、模块化等）
2. ❌ 禁止使用 JavaFX 等替代 GUI 框架
3. ❌ 禁止破坏 MVC 分层架构
4. ❌ 禁止在 EDT（事件调度线程）上执行耗时操作
5. ❌ 禁止擅自修改既定的配色/布局/交互规范
6. ❌ 禁止提交半成品、编译不通过、功能未验证的代码

## 二、需求分析

### 2.1 功能性需求全景

本项目共定义 **9 大功能模块**，覆盖图片压缩、视频压缩、文件管理、预览比较、配置持久化、主题系统等完整场景：

#### F1 文件导入（6 项）
- 单个/多个文件导入（JFileChooser）
- 整个文件夹导入（可选递归扫描）
- 从文件管理器拖拽文件到窗口
- 自动过滤不支持的文件格式并提示
- 自动去重（同一文件不重复添加）
- 单次导入上限 500 文件，防止 OOM，导入后按日期倒序排列

#### F2 文件列表管理（8 项）
- 显示文件名、原始大小、尺寸/时长、格式，图片显示缩略图
- 选中行高亮 + 缩略图预览
- 右键菜单（预览/打开原图/打开文件夹/单独压缩/单独设置参数/批量重命名/移除/清空）
- Delete 键移除选中文件 + Ctrl+Z 撤销
- 搜索/筛选/排序（按文件名/大小/原始顺序）
- 鼠标悬停 0.5 秒弹出 200×200 大图/视频首帧预览
- 逐文件参数标记（设置独立参数的 ⚙ 蓝色标记）
- 底部统计栏区分图片/视频数量 + 总大小

#### F3 图片压缩参数设置（9 项）
- 画质清晰度滑块（0-100），两端"体积更小← →更清晰"标注
- 4 个语义化快捷预设（网络分享 60 / 日常存档 80 / 摄影作品 95 / 极限压缩 30）
- 缩放模式：不缩放 / 按百分比 / 按最大尺寸 + ? 说明按钮 + 通俗说明文字 + 输出尺寸实时预览
- 输出格式：保持原格式 / JPEG / PNG / BMP + ? 格式说明弹窗
- 文件命名规则：5 种模式（后缀/前缀/保持原名/自定义文件名/基于图片信息 Token），支持 {name}/{width}/{height}/{sizeKB}/{counter} 等令牌
- 自定义输出目录 + 覆盖开关 + 保留 EXIF/XMP 开关
- 目标文件大小指定（二分搜索 8 轮迭代自动匹配质量）
- GIF 专属：最大颜色数控制（2-256 级通道色阶量化），仅选中 GIF 时显示
- 预估输出大小+输出尺寸实时显示

#### F4 图片压缩执行（7 项）
- 单张图片压缩（质量 + 缩放 + 格式转换 + EXIF 保留/移除 + 目标大小二分搜索）
- 批量图片压缩（多线程并行，2-4 线程池自适应 CPU 核心数）
- 实时进度反馈（ModernProgressBar 圆角渐变进度条 + 详情文字）
- 支持中途取消（Thread.interrupt() 机制）
- 单文件失败不影响整体（异常隔离）
- 无效参数组合检测（质量 100 + 不缩放 + 原格式 → 弹确认）
- 压缩大小守卫（输出>输入自动保留原文件 + Warning 提示）
- 逐文件独立压缩参数（PerFileCompressConfig + applyOverride 合并）

#### F5 预览与比较（7 项）
- 选中文件显示原图预览
- 压缩后自动切换效果图预览，支持缩放实时预览
- 三模式分段控件（原图 / 效果图 / 对比），选中段主题色实心填充
- 底部数据对比浮层（原始大小 / 压缩后 / 压缩率 / 尺寸）
- EXIF 信息展示面板（相机品牌/型号/拍摄时间/光圈/快门/ISO/焦距/GPS/图像尺寸/处理软件/版权）
- 棋盘格背景标识透明区域（主题感知）
- 预览面板操作按钮（打开原文件 / 打开输出目录）

#### F6 结果与通知（5 项）
- 压缩完成后弹出结果统计弹窗（成功数/失败数/总耗时/节省体积）
- Toast 通知（成功/错误/信息/警告 4 种类型，队列依次显示）
- 弹窗中点击"打开输出目录"在资源管理器中**高亮选中**输出文件
- 可勾选"完成后自动定位文件"保存偏好
- 失败文件列表标红 + 错误信息 Tooltip

#### F7 配置持久化（5 项）
- 自动保存压缩参数（质量/格式/命名/缩放/输出目录/GIF 颜色数/目标大小）
- 自动保存窗口状态（位置/大小/最大化/面板分割比例/当前模式）
- 自动保存主题选择 + 封面提取设置 + 自动定位偏好
- 损坏配置文件自动备份 + 重建默认（向后兼容新旧版本枚举值）
- 500ms 防抖写入，避免频繁磁盘 I/O

#### F8 主题系统（5 项）
- **23 套预设主题**：
  - 经典系列 7 套：默认蓝调/多巴胺暖/薄荷曼波/暖阳奶咖/极简墨灰/暗夜紫晶/深海蓝调
  - 蓝韵系列 6 套：蓝韵/海洋青/深邃黑蓝/落日橙/森林绿/薰衣草紫
  - 撞色系列 10 套：紫黑电竞/绿米自然/雾蓝冷感/嫩绿蓝清新/浅粉红暖意（各含浅深双模）
  - 遵循 60-30-10 法则 + WCAG 对比度标准
- 主题切换弹出面板（色块预览 + 名称）
- 一键切换无需重启（`FlatLaf.updateUI()` + ThemeChangeListener 自动刷新组件背景/前景/边框色）
- 主题自动记忆持久化
- 组件视觉统一：卡片 theme-primary 描边 + 按钮/输入框 PRIMARY 边框 + 分段控件选中态实心填充

#### F9 视频压缩（14 项）
- 图片/视频模式一键切换（双段分段控件，选中段主题色实心药丸）
- 视频文件导入（MP4/AVI/MOV/MKV/WebM/FLV/WMV/M4V）
- 视频元信息显示（时长/分辨率/帧率/编码/码率），基于 ffprobe JSON 解析
- 编码模式选择：CRF 画质优先 / 指定目标大小（码率模式自动计算最优码率）
- CRF 质量控制（滑块 0-100 显示映射到 CRF 0-51）+ 预估输出大小范围
- 3 套预设方案（微信小视频 CRF30+480p / B站投稿 CRF20+1080p+30fps / 存档原画 CRF15+原始）
- 分辨率缩放（原始/480p/720p/1080p/4K，自动偶数对齐）
- 帧率调整（保持原始/24/30/60 fps）
- 音频处理（保留/移除，编码器 H.264→AAC / WebM→Opus）
- 视频裁剪（起始时间 + 结束时间，FFmpeg `-ss` `-t` 参数）
- 输出格式选择（MP4 H.264 / WebM VP9 / AVI / MOV / MKV）+ ? 格式说明
- 封面快照提取（指定秒数截取帧，JPG/PNG 可选）
- 批量多版本导出（最多 20 个变体，每变体独立分辨率/帧率/CRF/音频/格式，自动描述性后缀命名）
- FFmpeg 自动检测（启动时自动搜索常见安装目录 + 捆绑版路径，不可用时降级提示）

## 三、架构设计

### 3.1 MVC 分层架构

```
                        App.java (入口)
              SwingUtilities.invokeLater()
           SplashScreen → Theme → MainFrame
                → Controller.init()
                            |
            ┌───────────────┼───────────────┐
            ▼               ▼               ▼
        View (Swing)    Controller       Model (POJO)
        MainFrame       MainController    ImageFileInfo
        FileListPanel   ImageController   VideoFileInfo
        PreviewPanel    VideoController   CompressConfig
        ParamPanel      WatchController   VideoCompressConfig
        VideoParamPanel                   CompressResult
            │               │               ▲
            │               ▼               │
            └─────────>  Service  <─────────┘
                        CompressService
                        VideoCompressService
                        BatchCompressService
                        FileManagerService
                        ConfigService
                        RenameService
                        FolderWatchService
                            │
                            ▼
                          Util
                     ImageCompressUtil / VideoCompressUtil
                     FileUtil / ImageUtil / ImageExifUtil
                     VideoUtil / FfmpegRenderUtil / VlcUtil
                     CoverExtractUtil / ThemeUtil / ThemePalette
                     CardWrapper / ValidateUtil / LogUtil
```

**依赖方向**（箭头 = "依赖于"）：
- **Model 层**：零依赖，纯数据模型（POJO + 枚举）
- **Util 层**：仅依赖 Model + JDK 标准库，无状态工具方法
- **Service 层**：依赖 Model + Util + 外部库，核心业务逻辑
- **Controller 层**：依赖 Service + Model + View，事件调度中心
- **View 层**：依赖 Controller + Model，纯界面绘制

### 3.3 核心数据流

```
用户拖拽文件
  → MainController.onDragImport()
  → FileManagerService.importFiles() [后台线程, 按日期倒序]
  → SwingUtilities.invokeLater()
  → FileListPanel.setFileList() [EDT]
  → 更新统计、启用压缩按钮

用户点击"开始压缩"
  → MainController.onStartCompress() [EDT]
  → 根据当前模式构建 CompressConfig / VideoCompressConfig
  → [逐文件参数] PerFileCompressConfig.applyOverride() 合并
  → CompressWorker.execute() [SwingWorker 后台线程]
  → BatchCompressService.compressBatch()
    → config.shallowCopy() 每文件独立配置副本 [防并行竞态]
    → [图片] CompressService → ImageCompressUtil → Thumbnailator
    → [视频] VideoCompressService → VideoCompressUtil → FFmpeg ProcessBuilder
  → worker.publish(ProgressChunk) → process() [EDT 更新 ModernProgressBar]
  → worker.done() → onCompressComplete() [EDT]
  → ResultDialog (含自动定位) + 预览更新 + Toast 通知
```

### 3.4 线程安全设计

| 场景 | 策略 | 说明 |
|------|------|------|
| 耗时操作 | `SwingWorker.doInBackground()` | 文件导入/压缩执行/EXIF 读取/FFmpeg 检测在后台线程 |
| UI 更新 | `SwingUtilities.invokeLater()` | 所有 Swing 组件操作回到 EDT |
| 进度报告 | `SwingWorker.publish()/process()` | 自动在 EDT 上批量处理进度块 |
| 批量压缩 | `ExecutorService` 线程池 | 图片 2-4 线程、视频 2 线程并行 |
| 并行竞态 | `CompressConfig.shallowCopy()` | 每文件创建独立配置副本，消除 TOCTOU |
| 配置文件 | 500ms Timer 防抖 | 滑块频繁拖动不会触发多次 I/O |
| FFmpeg 子进程 | `Process.waitFor()` + 超时 | 1 小时超时保护 + stdout 排空守护线程 |
| 取消操作 | `Thread.interrupt()` + `destroyForcibly()` | 双层保护确保线程和子进程均释放 |
| 主题切换 | `WeakHashMap<JComponent, Supplier<Color>>` + `fireThemeChange()` | 全局颜色刷新系统，自动适配所有动态颜色组件 |

## 五、关键技术难点与解决方案

### 5.1 Thumbnailator 不调用 scale/size 导致静默失败 ✅

**现象**：格式转换时抛出 `"size is not set"` 异常，仅 PNG→JPEG（经 alphaFill 路径）正常。

**原因**：Thumbnailator 0.4.20 的 `Builder` 在调用 `toFile()` 前必须至少调用一次 `scale()` 或 `size()` 方法。`ScaleMode.NONE` 的原始实现为空操作（no-op），不满足此约束。

**解决**：`ImageCompressUtil.applyScale()` 的 NONE/default 分支改为 `builder.scale(1.0)` 显式保持原尺寸。

### 5.2 Windows 文件锁定导致二次读取失败 ✅

**现象**：`ImageIO.read()` 读取尺寸后，`Thumbnails.of()` 再次打开同一文件时抛出 `IOException`。

**原因**：Windows 文件锁比 Linux 严格。`ImageIO.read()` 全量解码大图（~72MB）后可能未释放文件句柄。

**解决**：用 `ImageReader.setInput()` 仅读取 SOF 标记段获取宽高（~2KB），替代 `ImageIO.read()` 全量解码。

### 5.3 CMYK 色彩空间 JPEG 无法读取 ✅

**现象**：单反相机直出照片导入后尺寸显示 0×0，压缩失败。

**原因**：JDK 8 的 JPEG ImageReader 不支持 CMYK→RGB 色彩空间转换。

**解决**：改用 `ImageReader.setInput()` 读取 SOF 标记段（不涉及色彩空间解码），零尺寸时给出明确格式提示。

### 5.4 JDK 8 不支持 WebP 格式 ✅

**现象**：选择 WebP 输出格式后压缩静默失败。

**原因**：WebP 2010 年发布，JDK 8（2014 年）无 Reader/Writer。

**解决**：从 6 个文件中联动移除所有 WebP 引用。JDK 8 ImageIO 原生仅支持 JPG/PNG/BMP/GIF/WBMP。

### 5.5 EDT 阻塞导致启动画面卡死 ✅

**现象**：蓝色 SplashScreen 弹窗无法关闭。

**原因**：`Thread.sleep(300)` 在 EDT 上执行，Swing 单线程模型期间无法响应 UI 事件。

**解决**：三层防护 — `javax.swing.Timer` 替代 sleep + `try-finally` 异常保护 + 点击关闭兜底。

### 5.6 FlatLaf 主题切换后自定义组件颜色不更新 ✅

**现象**：`FlatLaf.updateUI()` 后自定义渲染组件文字颜色不变。

**原因**：自定义 `paintComponent()` 直接写死颜色常量，不从 UIManager 动态获取。

**解决**：建立 `WeakHashMap<JComponent, Supplier<Color>>` 动态颜色注册系统 — `setDynamicForeground()`/`setDynamicBackground()` 自动注册，`fireThemeChange()` 时全局刷新。解决了 #17（背景不更新）和后续的前景色更新问题，统一了所有组件的主题适配逻辑。

### 5.7 FFmpeg 不在系统 PATH 导致视频元数据全显示「未知」 ✅

**现象**：视频导入后元数据全部显示「未知」，即使 FFmpeg 已安装。

**原因**：仅查找系统 PATH 中的 `ffprobe`，未考虑自定义安装路径。

**解决**：实现多层 FFmpeg 自动发现 — JVM 属性 → PATH → 常见目录（winget/Chocolatey/Scoop）→ 捆绑版路径。

### 5.8 视频缩放后编码失败（奇数宽高） ✅

**现象**：竖屏视频缩放后 FFmpeg 报 "width not divisible by 2"。

**原因**：H.264/VP9 yuv420p 要求宽高均为偶数。`scale=W:-1` 计算可能得奇数。

**解决**：改用 `scale=W:-2`，`-2` 占位符自动计算并向下取偶数。

### 5.9 配置文件向后兼容（枚举版本迁移） ✅

**现象**：主题系统升级后旧 `config.json` 反序列化失败。

**原因**：Gson `valueOf("LIGHT")` 在枚举值移除后抛 `IllegalArgumentException`。

**解决**：注册自定义 `Gson TypeAdapter`，通过 `Theme.fromName()` 做新旧映射 + null 兜底默认主题。

### 5.10 ffplay stdout 管道死锁 ✅

**现象**：ffplay 播放时界面卡死。

**原因**：Java 端先读 stderr（阻塞等），ffplay stdout 满 → ffplay 阻塞在 write() → 死锁。

**解决**：stdout 排空放入独立 daemon 线程，主线程只读 stderr 解析进度。

### 5.11 图片目标大小二分搜索不收敛 ✅

**现象**：指定 500KB，实际输出 200KB 远小于目标。

**原因**：`bestQuality` 初始化为用户滑块值（如 80），二分搜索条件 `mid > bestQuality` 跳过所有低于 80 的 mid。试压缩异常失败时 `bestQuality` 保持 80。

**解决**：`bestQuality = 0`（替代滑块值）+ `!found` fallback + `shallowCopy()` 防竞态 + quality≥100 不达目标时 warning。

### 5.12 批量压缩并行 TOCTOU 竞态 ✅

**现象**：多线程批量压缩时 CUSTOM 命名模式所有文件同名，`generateUniqueFilename()` 竞态导致仅 2 个文件存活。

**原因**：所有线程共享同一个 `CompressConfig` 实例，`batchIndex` 字段并发修改产生竞态。

**解决**：`BatchCompressService` 中为每个文件调用 `config.shallowCopy()` 创建独立配置副本，各自的 `batchIndex` 独立递增。CUSTOM 模式无 `{counter}` Token 时自动追加序号兜底。

### 5.13 VBS 启动器 UTF-8 编码兼容性 ✅

**现象**：双击 `启动.vbs` 无法启动，报"路径未找到"运行时错误。命令行直接执行 Java 命令正常。

**原因**：VBScript 解析器对 UTF-8 编码（无 BOM）中的中文注释字符存在兼容性问题，即使注释本不该影响代码执行，也会导致后续 `FileExists()` 调用返回意外结果。

**解决**：VBS 完全重写为纯 ASCII（零中文注释），用 `Chr(34)` 替代 `""""` 多层引号转义消除 Windows 版本差异，添加 `Option Explicit` 和 `CurrentDirectory` 设置。

### 5.14 FlatLaf Component.borderColor 派生键被 updateUI 覆盖 ✅

**现象**：换肤后输入框/下拉框边框颜色不更新。

**原因**：`Component.borderColor` 是 FlatLaf 内部派生值（`shade(@background, 20%)`），`updateUI()` 时从背景色重新推导并覆盖手动 set 的值。

**解决**：完全放弃 UIManager 全局键方案，改为 `createDynamicLineBorder()` 工厂方法 — `paintBorder()` 时实时从 `ThemeUtil.PRIMARY` 读取颜色，无需缓存刷新。

## 八、个人收获与反思

### 8.1 技术收获

1. **MVC 架构从零实践**：完整搭建了 Model→Util→Service→Controller→View 五层分层项目，实现了控制器拆分（MainController 2057→624 行 -70%），理解了单向依赖的重要性。

2. **Swing 线程模型深度理解**：掌握了 EDT 单线程限制的底层原理，熟练使用 `SwingWorker` + `SwingUtilities.invokeLater()` + `javax.swing.Timer` 三类工具，避免了启动画面卡死、视频播放锁死、主题切换白屏等常见陷阱。

3. **FlatLaf 深度定制**：从 UIManager 全局属性到自定义 `paintComponent()` 与 LookAndFeel 动态绑定，实现了分段控件选中态实心填充、渐变进度条、棋盘格背景、PRIMARY 色描边、派生键覆盖策略等高级 UI 效果。特别是 `WeakHashMap<JComponent, Supplier<Color>>` 动态颜色注册系统，解决了大规模主题切换时的组件颜色同步问题。

4. **FFmpeg/FFprobe/FFplay 全流程集成**：实现了外部 CLI 调用、进度解析（stderr 正则）、管道死锁规避（stdout 排空 daemon 线程）、自动发现（多层路径扫描）、参数顺序语义（`-ss` 在 `-i` 前后区别）、奇数宽高约束（`-2` 占位符）。

5. **跨平台陷阱收集**：Windows 文件锁 vs Linux、CMYK vs RGB JPEG、JDK 8 格式限制、VLC 32/64 位 JNA 库名差异、UTF-8 VBS 解析器兼容性——每个问题都是实战经验的积累。

6. **配置向后兼容与并发安全**：Gson TypeAdapter 枚举版本迁移、`shallowCopy()` 解决并行 TOCTOU 竞态、损坏配置自动备份恢复。

### 8.2 工程收获

1. **原子提交规范**：156 次提交，每功能一提交，Conventional Commits 格式（`feat(scope):` / `fix(scope):` / `style(scope):` / `refactor(scope):`），可独立回溯任意历史版本。

2. **文档与代码同步**：进度文档条目数与 Git 提交一一对应，问题日志采用四段式（现象→原因→解决→教训），每个 Bug 都可追溯。

3. **分支管理实践**：6+ 功能分支独立开发，完成即合并，避免长期分支的合并冲突。

4. **免安装交付演进**：fat JAR → jlink 精简 JRE → 捆绑 JRE+FFmpeg+VLC → C 原生 EXE 启动器 v3（捆绑 JRE 优先）→ VBS 纯 ASCII 修复 → ZIP 打包，逐步演进到真正的"解压即用"。

5. **用户反馈驱动迭代**：从"边框不明显"调 alpha 值，到"vbs 不能启动"排查 UTF-8 编码，到"字都看不见了"建立全局颜色刷新系统——每条反馈都直接转化为代码改进。

### 8.3 遇到的最大挑战

| 挑战 | 难度 | 解决过程 |
|------|------|---------|
| ffplay 管道死锁 | ⭐⭐⭐⭐⭐ | 分析 ProcessBuilder 管道缓冲区和双向阻塞条件，独立线程排空 stdout |
| 图片目标大小二分搜索不收敛 | ⭐⭐⭐⭐⭐ | 研究 bestQuality 初始化逻辑、trial config 浅拷贝、fallback 兜底 |
| 批量压缩并行 TOCTOU 竞态 | ⭐⭐⭐⭐⭐ | 设计 shallowCopy() 每文件独立配置方案 |
| FlatLaf 派生键被 updateUI 覆盖 | ⭐⭐⭐⭐⭐ | 三次迭代：全局键→Component 键→动态 LineBorder 工厂方法 |
| EDT 阻塞排查（启动画面/播放/模式切换） | ⭐⭐⭐⭐ | 三处分别排查，用 Timer/SwingWorker/后台线程替换 |
| VLC 捆绑版无音频无控件 | ⭐⭐⭐⭐ | 三层防护：--plugin-path / VLC_PLUGIN_PATH / findBundledVlc() |
| 主题切换后自定义组件颜色不更新 | ⭐⭐⭐⭐ | 建立 WeakHashMap 全局动态颜色注册系统 |
| 视频裁剪 -ss 位置 + mergeWith 遗漏 | ⭐⭐⭐⭐ | 分析 FFmpeg seek 语义 + 审查所有工厂/合并方法 |
| Windows VLC 32/64 位兼容 | ⭐⭐⭐ | JNA 库名 libvlc vs vlc + 32/64 位路径差异 |
| VBS UTF-8 编码兼容性 | ⭐⭐⭐ | 验证所有路径正确后，通过二分法定位到编码问题 |
