# Java Swing 图片/视频压缩工具 — 项目实现计划

## Context

本项目是南昌航空大学软件工程专业暑期实训项目，目标是开发一款基于 Java Swing 的桌面图片/视频双模压缩工具。技术栈已确定（JDK 1.8 + Swing + FlatLaf + Thumbnailator + FFmpeg + Maven），不可更改。

本计划聚焦于**功能实现顺序**和**模块依赖关系**，将原始规划书转化为可逐日执行的任务清单，确保依赖正确的构建顺序，避免返工。

---

## 一、改进建议

基于原始规划书，建议以下调整：

| # | 改进点 | 原方案 | 新方案 | 理由 |
|---|--------|--------|--------|------|
| 1 | 包名 | `com.compress` | `com.nchu.imagecompress` | 遵循反向域名规范，避免与第三方库冲突 |
| 2 | 配置持久化 | 未明确 | JSON 文件 (Gson)，存于 `~/.NCHUImageCompressor/config.json` | 支持嵌套结构，比 Properties 文件更灵活；比 Windows 注册表更可迁移 |
| 3 | 进度回调 | 未定义接口 | `ProgressListener` 接口 (Service → Controller) | 明确的跨层契约，Service 层不引入 Swing 依赖 |
| 4 | UI 构建策略 | 逐一完整构建面板 | **骨架先行**：先搭所有面板的空布局，验证无误后再逐个填充功能 | 布局问题早期暴露，减少后期大面积返工 |
| 5 | 打包方案 | JLink + Launch4j | 编译用 JDK 8，打包用 JDK 17 的 jlink 生成精简 JRE | JLink 需要 JDK 9+，JDK 8 不包含此工具 |
| 6 | 易用性优化 | 保存上次参数 | 增加**智能质量推荐**（照片/截图/图标三类）、拖拽排序、撤销移除 | 提升答辩演示的差异化亮点 |
| 7 | 开发周期 | 14 天笼统划分 | 14 天按**依赖拓扑排序**精确到半天的任务 | 每天有明确交付物，可追踪进度 |
| 8 | 视频压缩 | 无 | 新增视频压缩功能（FFmpeg CLI 集成） | 扩展项目能力边界，提升答辩差异化亮点 |

## 二、模块依赖图（决定构建顺序）

```
                          App.java (入口)
                              |
              +---------------+---------------+
              |               |               |
              v               v               v
         View (Swing)    Controller      Model (POJO)
              |               |               ^
              |               v               |
              +-------->  Service  <---------+
                              |
                              v
                           Util
                              |
                              v
              外部库 (Thumbnailator, FlatLaf, Gson, FFmpeg CLI)
```

**编译依赖规则**（箭头 = "依赖于"）：
- **Model**：零依赖，必须最先构建（含 ImageFileInfo / VideoFileInfo）
- **Util**：仅依赖 Model + JDK 标准库，第二批构建（含 ImageUtil / VideoUtil）
- **Service**：依赖 Model + Util + 外部库，第三批构建（最核心的业务层，含 ImageCompressUtil / VideoCompressUtil）
- **Controller**：依赖 Service + Model，第四批构建
- **View**：依赖 Controller + Model，第五批构建（含 图片/视频模式切换）
- **App**：组装一切，最后构建

## 三、跨层关键接口（必须先定义）

### 3.1 进度回调接口（Service → Controller）

```java
// service/ProgressListener.java
public interface ProgressListener {
    void onFileStarted(int index, int total, String fileName);
    void onFileProgress(int index, String fileName, double percent);
    void onFileCompleted(int index, CompressResult result);
    void onBatchCompleted(List<CompressResult> results, int success, int fail, long elapsedMs);
    void onBatchCancelled();
}
```

### 3.2 视图回调接口（View → Controller）

```java
// controller/MainControllerCallback.java
public interface MainControllerCallback {
    void onImportFiles(List<File> files);
    void onImportFolder(File folder);
    void onRemoveFiles(List<Integer> indices);
    void onClearAllFiles();
    void onStartCompress(CompressConfig config);
    void onCancelCompress();
    void onSelectFile(int index);
    void onToggleTheme();
    void onWindowClosing();
}
```

### 3.3 核心数据模型

```
ImageFileInfo    — 文件路径、原始大小、宽高、格式、处理状态、缩略图
CompressConfig   — 质量(0-100)、缩放模式/比例、输出格式、输出路径、命名规则
CompressResult   — 输入信息、输出文件、压缩前后大小、压缩率、成功/失败、耗时
AppConfig        — 主题、窗口位置/大小、上次参数、最近目录（可序列化为 JSON）
VideoFileInfo    — 视频路径、时长、分辨率、帧率、码率、编码格式、处理状态
VideoCompressConfig — CRF/码率、分辨率模式、输出格式(MP4/WebM/MKV)、帧率、音频选项、裁剪起止时间
```

## 六、技术风险与缓解

| 风险 | 严重度 | 缓解措施 |
|------|--------|---------|
| FlatLaf 主题切换导致自定义控件样式错乱 | 中 | 第 5 天尽早测试主题切换，优先使用 FlatLaf 标准控件 |
| JLink 需要 JDK 9+，JDK 8 不包含 | 高 | 编译用 JDK 8，打包用 JDK 17 的 jlink；或退而求其次只发布 fat JAR |
| 大图片(50MB+)预览 OOM | 中 | 预览统一缩放到 1024px 以内；使用 Thumbnailator 流式读取 |
| 批量压缩界面假死 | 高 | 严格使用 SwingWorker，所有 UI 更新通过 EDT；进度更新节流(100ms) |
| 文件覆盖丢失原图 | 高 | 默认添加后缀，重名自动编号；覆盖选项默认不勾选 |
| FFmpeg 未安装或版本过低 | 高 | 启动时检测 FFmpeg 可用性，不可用时视频模式降级提示；捆绑 FFmpeg 到免安装版 |
| ffprobe JSON 解析失败（编码/格式差异） | 中 | Jackson 容错解析 + try-catch 兜底，失败时用正则提取关键字段 |
| 视频压缩耗时极长（大文件） | 高 | SwingWorker 后台执行 + 进度实时更新 + 可取消；预设 CRF 值防用户误设极端值 |
| ProcessBuilder 子进程残留 | 中 | 取消时 `Process.destroy()` + `waitFor()` 确保子进程退出 |

## 七、验证方案

### 功能验证（核心流程）
1. 启动软件 → 看到启动页 → 进入主界面
2. 拖拽 5 张图片到窗口 → 列表显示 5 条记录（含文件名、大小、尺寸）
3. 调整质量滑块到 60% → 选择 JPG 格式 → 点击压缩
4. 压缩完成 → 预览区自动切换到效果图 → 对比数据显示压缩率
5. 批量导入 20 张 → 一键压缩 → 进度条实时更新 → 结果弹窗显示统计
6. 切换暗色主题 → 所有控件配色同步 → 无样式错乱
7. 关闭软件 → 重新打开 → 上次参数和主题自动恢复
8. 双击 EXE（无 JDK 环境）→ 软件正常运行

### 视频功能验证
1. 切换至「视频压缩」模式 → 参数面板切换为视频参数
2. 导入 MP4/AVI/MOV/MKV 视频文件 → 列表显示视频信息（🎬 + 时长）
3. 选择 CRF 预设 → 点击压缩 → FFmpeg 调用 → 进度更新
4. 压缩完成 → 结果弹窗显示统计（节省空间 + 耗时）
5. 无 FFmpeg 环境 → 视频模式提示"FFmpeg 未安装"
6. 批量视频压缩 → 中途取消 → 已完成文件保留

### 异常验证
- 导入损坏图片 → 标记失败，不影响其他文件
- 选择系统保护目录为输出路径 → 提示权限不足
- 批量压缩中点击取消 → 任务终止，已压缩文件保留
- 输入负数质量 → 滑块限制在 0-100，无法输入非法值
