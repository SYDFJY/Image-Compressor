## 1. 简介

**NCHU Compressor** 是一款基于 Java Swing 的**图片/视频双模式桌面压缩工具**。它提供了直观的拖拽式操作界面，支持图片和视频两种压缩模式，可在参数面板中自由调整质量、分辨率、格式等参数，并实时预览压缩效果。

### 核心特性

- **双模式切换**：工具栏分段控件一键切换图片压缩 / 视频压缩模式
- **图片压缩**：质量调节、尺寸缩放、格式转换、目标大小（二分搜索）、GIF 颜色优化、EXIF 信息查看
- **视频压缩**：CRF 画质控制、码率模式（指定目标大小）、分辨率调整、帧率设置、音频保留/移除、视频裁剪、多版本批量导出、封面快照提取
- **实时预览**：图片原图/效果图/对比三种预览模式 + 缩放实时预览；视频 VLCJ 内嵌播放（降级 FFmpeg 管道帧渲染）
- **智能命名**：多种命名规则 + 自定义文件名 + 基于图片信息自动命名（分辨率/大小等 Token）
- **批量处理**：多文件并行压缩，单文件失败不影响整体
- **批量重命名**：源文件 Token 模板批量重命名（{date}/{counter}/{name} 等）
- **逐文件参数**：每个文件可单独设置压缩参数，批量压缩不强制统一
- **文件夹监控**：自动监控指定文件夹，新文件自动按预设参数压缩（懒人模式）
- **悬停预览**：鼠标悬停 0.5 秒弹出 200×200 大图/视频首帧预览
- **压缩后自动定位**：完成压缩后在资源管理器中高亮选中输出文件
- **23 套主题**：经典系列 + 蓝韵系列 + 撞色系列，浅色/深色全覆盖
- **免安装版**：Windows 便携包（JRE + FFmpeg + VLC 全捆绑），解压即用
- **参数记忆**：关闭软件后自动保存所有设置，下次启动恢复

## 2. 安装与环境要求

### 2.1 方式一：免安装版（Windows，推荐）

下载 `NCHU-Compressor-v1.0.0-免安装版.zip`，解压到任意文件夹，双击 `image-compressor.exe`（或 `启动.vbs`）即可运行。

```
NCHU Compressor 免安装版/
├── image-compressor.exe        ← 双击启动（自动使用捆绑 JRE）
├── 启动.vbs                    ← 备选：无控制台静默启动
├── 启动.bat                    ← 备选：带控制台（可查看日志）
├── image-compressor-1.0.0.jar  ← 主程序
├── jre/                        ← Java 8 运行时（已捆绑）
├── ffmpeg/bin/                 ← FFmpeg（已捆绑）
└── vlc/                        ← VLC 原生库（已捆绑）
```

> **无需安装任何东西** — Java、FFmpeg、VLC 全部内置，打开即用。

### 2.2 方式二：JAR 运行（跨平台）

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| **Java Runtime (JRE)** | 1.8 或更高 | 运行 Java 程序的基础环境 |
| **操作系统** | Windows 7+ / macOS 10.12+ / Linux | 64 位系统推荐 |

> **如何检查 Java 版本**：在终端执行 `java -version`，如显示 `1.8.0_xxx` 或更高即可。

### 2.3 视频压缩环境（可选）

视频压缩功能需要 **FFmpeg** 外部工具。如果只使用图片压缩，无需安装。免安装版已内置 FFmpeg。

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| **FFmpeg** | 4.0 或更高 | 视频编解码核心工具 |

**安装 FFmpeg**：

- **Windows**：下载 [ffmpeg.org](https://ffmpeg.org/download.html) 的 Windows 构建版，解压后将 `bin` 目录添加到系统 PATH 环境变量；或使用 `winget install ffmpeg`
- **macOS**：`brew install ffmpeg`
- **Linux**：`sudo apt install ffmpeg`（Ubuntu/Debian）或 `sudo dnf install ffmpeg`（Fedora）

软件启动时会**自动搜索**以下位置来发现 FFmpeg：

- `FFMPEG_HOME` 环境变量
- `~/.ffmpeg/bin`
- `%LOCALAPPDATA%/ffmpeg/bin`（Windows）
- `C:\ffmpeg\bin`（Windows）
- Chocolatey / Scoop 安装目录（Windows）
- **同目录下的 `ffmpeg/bin/`（免安装版捆绑）**
- 系统 PATH

也可以在「设置」对话框中手动指定 FFmpeg 路径。

### 2.4 视频预览环境（可选）

视频**预览播放**需要 VLC 播放器。没有 VLC 时，软件会用 FFmpeg 逐帧渲染（性能较低）。免安装版已内置 VLC。

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| **VLC Media Player** | 3.0 或更高 | 视频预览播放引擎 |

- 下载地址：[videolan.org](https://www.videolan.org/)

### 2.5 启动软件

**免安装版（Windows）**：
```
双击 image-compressor.exe 或 启动.vbs
```

**JAR 运行（跨平台）**：
```bash
java -jar image-compressor-1.0.0.jar
```

## 4. 快速上手

### 4.1 图片压缩 — 3 步完成

**第 1 步：导入图片**

点击工具栏「导入文件」按钮，选择一个或多个图片文件。也可以直接从文件管理器**拖拽图片**到左侧文件列表区域。

**第 2 步：调整参数**

在右下方的参数面板中，拖动「画质清晰度」滑块——**往左文件更小、往右画质更高**。默认值 80 适用于大多数场景。参数面板底部会实时显示**预估输出大小**和**输出尺寸**。

**第 3 步：开始压缩**

点击参数面板底部的蓝色「开始压缩」按钮（或按 `Ctrl+Enter`）。压缩完成后会弹出结果窗口，可以点击「打开输出目录」直接在资源管理器中定位压缩后的文件。

---

### 4.2 视频压缩 — 3 步完成

**第 1 步：切换到视频模式**

点击工具栏分段控件中的「**视频**」按钮，右侧面板切换为视频压缩界面。

**第 2 步：导入视频**

点击「导入文件」选择一个视频文件（支持 MP4、AVI、MOV、MKV 等格式）。

**第 3 步：调整参数并压缩**

拖动「视频清晰度」滑块调整画质（或在编码模式中选择「指定目标大小」），选择分辨率和输出格式，点击「开始压缩」。

### 7.2 基础设置 — 参数详解

#### 7.2.1 画质清晰度

| 属性 | 说明 |
|------|------|
| **中文名** | 画质清晰度 |
| **控件** | 水平滑块 + 百分比数字标签，两端标注「体积更小 ←」「→ 更清晰」 |
| **取值范围** | 0 ~ 100 |
| **默认值** | **80** |
| **单位** | %（百分比） |

**作用**：控制图片压缩时的质量级别。

- **数值越高（如 90↑）**：画质越好，保留更多细节，但**文件更大**
- **数值越低（如 50↓）**：文件更小，但会出现模糊、色块等画质损失
- **数值 100**：无损输出（仅限格式转换场景，JPEG 格式下仍会有轻微压缩）

**适用场景建议**：

| 场景 | 推荐质量 | 说明 |
|------|---------|------|
| 网络分享 | 60 - 70 | 兼顾清晰度与文件大小 |
| 日常存档 | 75 - 85 | 默认值 80 适合大多数图片 |
| 摄影作品 | 90 - 95 | 保留丰富细节 |
| 极限压缩 | 10 - 40 | 文件极小但画质明显下降 |

**预估输出大小**：滑块下方实时显示预估输出文件大小，帮你直观了解参数调整对文件体积的影响。

> **原理**：JPEG 压缩通过丢弃人眼不敏感的色彩细节来减小体积。质量参数本质上是 JPEG 量化表的缩放因子——数值越低，丢弃的信息越多。

---

#### 7.2.2 快捷预设

| 预设 | 质量值 | 适用场景 |
|------|--------|---------|
| **网络分享** | 60 | 兼顾清晰度与文件大小，适合微信/QQ 发送 |
| **日常存档** | 80 | 日常使用，画质与体积最佳平衡 |
| **摄影作品** | 95 | 追求高画质，适合打印/收藏 |
| **极限压缩** | 30 | 追求最小体积，画质有明显损失 |

点击预设按钮会自动设置对应的质量值，选中态以主题色实心药丸样式高亮。

---

#### 7.2.3 缩放模式

| 属性 | 说明 |
|------|------|
| **中文名** | 缩放模式 |
| **控件** | 下拉选择框 + `?` 说明按钮 |
| **选项** | 不缩放 / 按百分比 / 按最大尺寸 |
| **默认值** | **不缩放** |

**三种模式详解**：

| 模式 | 行为 | 适用场景 |
|------|------|---------|
| **不缩放** | 保持原始宽高，不做任何缩放处理 | 只想降低质量减小体积，不需要改尺寸 |
| **按百分比** | 将宽和高同时缩小到原始尺寸的指定比例 | 需要精确控制缩小比例，如缩小到 50% |
| **按最大尺寸** | 将图片等比缩放到指定的最大宽高范围内，**保持宽高比** | 需要限制输出图片的最大尺寸，如「宽度不超过 1920」 |

缩放模式下方有**通俗说明文字**和**输出尺寸实时预览**（如「输出尺寸：1920 × 1080（50%）」），帮你理解当前设置的缩放效果。

**按百分比 — 子参数**：

| 属性 | 说明 |
|------|------|
| **中文名** | 缩放比例 (%) |
| **控件** | 数字输入框（带上下箭头） |
| **取值范围** | 1 ~ 100 |
| **默认值** | **100** |

- 设为 50 表示输出图片宽高各缩小为原来的 50%，面积变为原来的 25%
- 设为 100 等同于不缩放

**按最大尺寸 — 子参数**：

| 属性 | 说明 |
|------|------|
| **参数** | 最大宽度（像素）、最大高度（像素） |
| **默认值** | 宽 **1920**、高 **1080**（Full HD） |
| **单位** | px（像素） |

- 图片等比缩放，确保宽度 ≤ 最大宽度 **且** 高度 ≤ 最大高度
- 例如：原图 4000×3000，限制 1920×1080 → 输出 1440×1080（按高度限制作等比缩放）
- 如果原图本身就小于限制尺寸，则不会放大

### 7.3 高级设置 — 参数详解

#### 7.3.1 文件命名

| 属性 | 说明 |
|------|------|

| **默认值** | **添加后缀 `_compressed`** |

**五种规则详解**：

| 规则 | 输出文件名示例 | 注意事项 |
|------|--------------|---------|
| **添加后缀** | `DSC00123.jpg` → `DSC00123_compressed.jpg` | 后缀可在下拉框中自定义 |
| **添加前缀** | `DSC00123.jpg` → `compressed_DSC00123.jpg` | 前缀可在下拉框中自定义 |

**基于图片信息模式的 Token 说明**：

| Token | 含义 | 示例 |
|-------|------|------|
| `{name}` | 原文件名（不含扩展名） | `DSC00123` |
| `{width}` | 图片宽度（像素） | `4000` |
| `{height}` | 图片高度（像素） | `3000` |
| `{size}` | 文件大小（字节） | `2456789` |
| `{sizeKB}` | 文件大小（KB） | `2399` |
| `{format}` | 输出格式扩展名 | `jpg` |
| `{quality}` | 压缩质量值 | `80` |
| `{counter}` | 自动序号（1-based） | `1` |
| `{counter:N}` | 补零序号（N 位） | `{counter:3}` → `001` |

> 默认模板：`{name}_{width}x{height}`，输出如 `DSC00123_4000x3000.jpg`

### 8.2 视频参数 — 参数详解

#### 8.2.1 编码模式

| 属性 | 说明 |
|------|------|
| **中文名** | 编码模式 |
| **控件** | 下拉选择框 |
| **选项** | CRF 画质优先 / 指定目标大小（码率模式） |
| **默认值** | **CRF 画质优先** |

- **CRF 画质优先**：使用 CRF 恒定质量编码，画质稳定但无法精确控制文件大小
- **指定目标大小**：根据目标文件大小（MB）自动计算最优码率，使用 `-b:v` 码率控制模式

---

#### 8.2.2 视频清晰度（CRF 模式）

| 属性 | 说明 |
|------|------|
| **中文名** | 视频清晰度 |
| **控件** | 水平滑块，两端标注「体积更小 ←」「→ 更清晰」 |
| **显示范围** | 0 ~ 100（显示值） |
| **实际范围** | CRF 0 ~ 51 |
| **默认值** | **55（显示值）= CRF 23** |
| **映射关系** | CRF = round((100 - 显示值) × 51 / 100) |

**作用**：CRF（Constant Rate Factor，恒定质量因子）是 H.264/H.265 编码器最重要的画质参数。

- **显示值越高（CRF 越低）**：画质越好，但**文件越大**
- **显示值越低（CRF 越高）**：文件越小，但画质越差

| 显示值 | 实际 CRF | 画质描述 | 适用场景 |
|--------|---------|---------|---------|
| 80 - 100 | 0 - 10 | 近乎无损，视觉无差异 | 专业存档、母带备份 |
| 60 - 80 | 10 - 20 | 高质量，肉眼难辨差异 | B站投稿、高清收藏 |
| 40 - 60 | 20 - 30 | 中等质量，轻微压缩痕迹 | **日常使用默认（55/CRF23）** |
| 20 - 40 | 30 - 40 | 低质量，可见压缩痕迹 | 微信传输、快速预览 |
| 0 - 20 | 40 - 51 | 明显画质损失 | 极限压缩 |

**预估输出大小**：滑块下方实时显示预估文件大小范围（查表法，根据 CRF 级别估算百分比区间）。

> **原理**：CRF 编码器会动态分配码率——复杂场景用更多数据、简单场景用更少数据，在保持恒定视觉质量的同时尽量减小体积。

## 9. 批量处理

### 9.1 批量压缩工作流

1. 导入多个文件到文件列表
2. 设置压缩参数（所有文件使用相同全局参数，或为特定文件设置逐文件参数）
3. 点击「开始压缩」
4. 后台并行处理（图片：2-4 线程；视频：2 线程）
5. 实时显示进度（状态栏 ModernProgressBar + 文件状态更新）
6. 完成后弹出结果统计窗口

### 9.2 并行处理

- **图片压缩**：最多使用 **4 个线程**同时处理（自动检测 CPU 核心数，钳制在 2~4）
- **视频压缩**：最多使用 **2 个线程**（视频压缩更耗资源）

### 9.3 容错机制

- 单个文件压缩失败**不影响**其他文件的处理
- 失败的文件在列表中显示「❌ 失败」状态
- 结果窗口汇总：成功数、失败数、节省空间、压缩率、总耗时
- 对于 INFO_BASED 和 CUSTOM 命名模式，自动为每个文件生成独立配置副本（`shallowCopy`），防止并行竞态

### 9.4 取消压缩

- 按 `Escape` 键或点击「取消」按钮
- 图片：中断 SwingWorker 后台线程
- 视频：强制终止 FFmpeg 进程

### 9.5 压缩结果窗口

压缩完成后弹出统计窗口：

| 信息 | 说明 |
|------|------|
| **成功** | 成功压缩的文件数 |
| **失败** | 压缩失败的文件数 |
| **节省空间** | 原始总大小 − 压缩后总大小 |
| **压缩率** | 节省空间 ÷ 原始总大小 × 100% |
| **总耗时** | 从点击压缩到完成的用时 |

窗口底部有「打开输出目录」按钮——点击直接在资源管理器中**高亮选中**第一个成功输出的文件，无需手动查找。可勾选「完成后自动定位文件」记忆偏好。

> **安全保护**：如果压缩后体积反而比原文件大，软件会自动保留原文件并给出警告提示。

### 9.6 逐文件压缩参数

右键菜单「⚙ 单独设置此文件参数…」可以为特定文件设置独立的压缩参数。设置了单独参数的文件在列表中显示 ⚙ 蓝色标记。批量压缩时，每个文件使用自己的参数（如有）或全局参数（如无）。

## 15. 常见问题

### 15.1 免安装版相关

**Q：免安装版打不开 / 双击 EXE 没反应？**

> 1. 确认解压后文件夹结构完整（`jre/`、`ffmpeg/bin/`、`vlc/` 与 EXE 同级）
> 2. 尝试右键 `启动.vbs` → 用记事本打开，确认 `javaw.exe` 路径
> 3. 某些杀毒软件可能拦截，尝试添加信任

**Q：免安装版体积为什么这么大？**

> 解压后约 535MB（JRE 117MB + FFmpeg 414MB + VLC 128MB + JAR 2MB）。这是完整独立运行环境的代价——换来的是用户无需安装任何东西。

### 15.2 图片相关

**Q：为什么我的图片无法导入 / 提示"格式不支持"？**

> 支持的图片格式：JPG、JPEG、PNG、BMP、GIF、TIFF、TIF、ICO、WebP。  
> 已知不支持的格式：PSD、AI、EPS、SVG、RAW（CR2/NEF/ARW/DNG 等）。  
> 如果 CMYK 色彩空间的 JPEG 无法解析，请用画图/Photoshop 转为 RGB 模式后重试。

**Q：压缩后图片变大是怎么回事？**

> 当原图已经被高度压缩过时（如网络下载的图片），再次压缩可能反而变大。软件会自动检测并保留原文件。

**Q：PNG 转 JPEG 后背景变黑了？**

> 已自动处理此问题——PNG 转 JPEG/BMP 时透明区域自动填充为白色。

### 15.3 视频相关

**Q：提示"FFmpeg 未找到"怎么办？**

> 1. 免安装版用户：确保 `ffmpeg/bin/` 文件夹与 EXE 同级
> 2. JAR 用户：确认已安装 FFmpeg（终端执行 `ffmpeg -version` 验证）
> 3. 如果已安装但软件找不到，在「编辑 → 设置」中手动指定 FFmpeg 路径

**Q：视频预览无法播放？**

> 免安装版已内置 VLC，启动器自动配置路径。  
> JAR 运行用户：视频预览需要 VLC 播放器。安装 VLC 后重新启动即可。没有 VLC 时软件会用 FFmpeg 逐帧渲染（性能较低，无音频）。

**Q：CRF 值和"清晰度"滑块的关系？**

> 清晰度滑块显示 0-100（越高越好），内部映射到 CRF 0-51（越低越好）。映射公式：CRF = (100 − 显示值) × 51 ÷ 100。例如显示值 55 → CRF ≈ 23。

**Q：视频压缩很慢？**

> 视频压缩是计算密集型任务。以下因素影响速度：
> - **分辨率越高**（4K > 1080p > 720p）越慢
> - **CRF 越低**（画质越高）编码越慢
> - **帧率越高**处理量越大
> - CPU 性能直接影响处理速度
>
> 建议：选择合适的画质而不追求极致低 CRF，1080p 一般是性价比最高的分辨率。

### 15.4 通用

**Q：如何重置所有设置？**

> 删除配置文件 `~/.NCHUImageCompressor/config.json`，下次启动将恢复默认设置。

**Q：软件占用内存很大？**

> 处理超大图片时可能出现内存不足。可以调大 JVM 堆内存：
> ```bash
> java -Xmx2048m -jar image-compressor-1.0.0.jar
> ```
> 其中 `2048m` 表示分配 2GB 内存。免安装版用户可编辑 `启动.bat` 添加 `-Xmx2048m` 参数。

**Q：模式切换后图片列表不见了？**

> 这是正常的模式隔离设计——图片和视频模式各自维护独立的文件列表。切换回图片模式后列表会恢复。
