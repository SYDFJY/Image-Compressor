## 一、项目概述

### 1.1 项目背景

在日常学习与工作中，图片文件体积过大导致传输慢、存储占用多、网页加载延迟等问题频繁出现。市面上的在线压缩工具存在隐私泄露风险，专业软件（如 Photoshop）操作门槛高。本项目开发一款**离线、轻量、易用**的桌面图片压缩工具，解决上述痛点。

### 1.2 项目目标

| 目标 | 描述 |
|------|------|
| **核心功能** | 支持 JPG/PNG/BMP/GIF/TIFF/ICO 格式的批量图片压缩，含质量调节、尺寸缩放、格式转换 |
| **易用性** | 拖拽导入、右键菜单、全局快捷键、参数记忆、3 步完成压缩 |
| **用户体验** | 启动画面、Toast 通知、进度反馈、预览对比、6 套可切换主题 |
| **可靠性** | 单文件失败不影响整体、中途可取消、异常全面容错 |
| **可交付** | 绿色免安装 fat JAR + Windows 启动器 + 完整测试清单 |

### 1.3 技术栈

| 层级 | 技术 | 版本 | 选型理由 |
|------|------|------|---------|
| 语言 | Java | JDK 1.8 | 实训课程要求，跨平台兼容性最好 |
| GUI | Java Swing | JDK 原生 | 无需额外运行时，JDK 内置 |
| 主题 | FlatLaf + Extras | 3.2.5 | 现代化 LookAndFeel，支持圆角/阴影/字体全局配置 |
| 图像处理 | Thumbnailator | 0.4.20 | 链式 API，单行代码完成缩放+质量+格式转换 |
| 配置存储 | Gson | 2.8.9 | Google 出品 JSON 库，轻量无依赖 |
| 构建 | Maven | 3.9.6 | 标准 Java 项目构建工具 |
| 测试 | JUnit | 4.13.2 | 单元测试框架 |
| 版本管理 | Git | — | 本地 + GitHub 远程仓库 |

### 3.3 核心数据流

```
用户拖拽文件
    → MainController.onDragImport()
    → FileManagerService.importFiles() [后台线程]
    → SwingUtilities.invokeLater()
    → FileListPanel.addFile() [EDT]
    → 更新统计、启用压缩按钮

用户点击"开始压缩"
    → MainController.onStartCompress() [EDT]
    → 构建 CompressConfig
    → CompressWorker.execute() [后台线程]
    → BatchCompressService.compressBatch()
        → CompressService.compress()
            → ImageCompressUtil.compress() → Thumbnailator
    → worker.publish(ProgressChunk) → process() [EDT]
    → worker.done() → onCompressComplete() [EDT]
    → ResultDialog.show() + 预览更新 + Toast
```

## 五、关键技术挑战与解决方案

### 5.1 Thumbnailator 不调用 scale/size 导致静默失败

**现象**：格式转换时报 `"size is not set"` 异常，仅 PNG→JPEG（经 alphaFill 路径）正常。

**原因**：Thumbnailator 0.4.20 的 `Builder` 在调用 `toFile()` 前必须至少调用一次 `scale()` 或 `size()`。`ScaleMode.NONE` 时 `applyScale()` 为空操作。

**解决**：`builder.scale(1.0)` 显式保持原尺寸。

### 5.2 Windows 文件锁定导致二次读取失败

**现象**：`ImageIO.read()` 读尺寸后 `Thumbnails.of()` 再次打开同一文件抛异常。

**原因**：Windows 文件锁比 Linux 严格，`ImageIO.read()` 全量解码后可能未释放句柄。

**解决**：用 `ImageReader.setInput()` 仅读 SOF 元数据（~2KB），替代全量解码（~72MB）。同时解决了 CMYK JPEG 解码失败问题。

### 5.3 CMYK 色彩空间 JPEG 无法读取

**现象**：单反直出照片导入后尺寸显示 0×0。

**原因**：JDK 8 ImageIO 不支持 CMYK→RGB 转换，`ImageIO.read()` 返回 null。

**解决**：改用 `ImageReader` 读 SOF 标记（标记段不涉及色彩空间解码），零尺寸时给明确提示。

### 5.4 JDK 8 不支持 WebP 格式

**现象**：选择 WebP 输出格式后压缩失败。

**原因**：WebP 2010 年发布，JDK 8（2014 年）的 ImageIO 无 Reader/Writer。JDK 15+ 才原生支持。

**解决**：从 6 个文件（枚举/下拉框/过滤器/配置恢复/文件工具/压缩工具）中联动移除所有 WebP 引用。

### 5.5 EDT 阻塞导致启动画面卡死

**现象**：蓝色 SplashScreen 弹窗无法关闭，只能强杀进程。

**原因**：`Thread.sleep(300)` 在 EDT 上执行，Swing 单线程模型下 sleep 期间无法响应任何 UI 事件。异常未捕获时 `dispose()` 永远不执行。

**解决**：三层防护——(1) `javax.swing.Timer` 替代 sleep；(2) `try-finally` 确保异常时 dispose；(3) 点击画面手动关闭兜底。

### 5.6 配置文件向后兼容

**现象**：多主题系统升级后，旧 `config.json` 中 `"theme": "LIGHT"` 无法反序列化。

**原因**：Gson 调用 `Theme.valueOf("LIGHT")` 但新枚举无此值。

**解决**：注册自定义 `TypeAdapter`，用 `Theme.fromName()` 做新旧映射（LIGHT/DARK → BLUE_CLASSIC/DARK_PURPLE），外加 `getTheme()` null 兜底。

## 八、个人收获与反思

### 8.1 技术收获

1. **MVC 架构实操**：从零搭建分层项目，理解依赖方向（Model←Util←Service←Controller→View）
2. **Swing 线程模型**：EDT 单线程限制 → `SwingWorker` + `SwingUtilities.invokeLater()` + `javax.swing.Timer`
3. **FlatLaf 深度定制**：UIManager 全局属性 + 自定义 LookAndFeel + `updateComponentTreeUI()` 动态刷新
4. **Thumbnailator 踩坑**：Builder 模式必须显式调 `scale()`/`size()`，`ImageIO.read()` 与 `Thumbnails.of()` 的文件锁冲突
5. **Gson 向后兼容**：自定义 TypeAdapter 处理枚举变更
6. **跨平台陷阱**：Windows 文件锁 vs Linux、CMYK vs RGB、JDK 8 格式限制

### 8.2 工程收获

1. **原子提交**：24 次提交，每功能一提交，可独立回溯
